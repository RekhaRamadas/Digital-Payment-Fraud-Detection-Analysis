# Digital-Payment-Fraud-Detection-Analysis
# Project Overview

This project focuses on analysing digital payment transactions to identify fraudulent activities using Python. The analysis includes data cleaning, exploratory data analysis (EDA), and data visualization to discover fraud patterns, customer behaviour, transaction trends, and financial risk indicators.

The project demonstrates how data analytics can help financial institutions detect suspicious transactions and improve fraud prevention strategies.
## Project Objectives

- Perform data cleaning and preprocessing.
- Explore transaction patterns using EDA.
- Visualize fraud trends using different charts.
- Identify important fraud risk factors.
- Generate business insights and recommendations.

## Dataset

The dataset contains digital payment transaction records with customer, merchant, transaction, and fraud-related information.

- ## Key Findings

- Fraudulent transactions represent only a small percentage of the dataset.
- Mobile and Card transactions contribute significantly to fraud cases.
- High merchant risk scores are associated with fraudulent activities.
- Higher IP risk scores increase fraud probability.
- Customers with frequent transactions show relatively higher fraud risk.
- High-value transactions require additional monitoring.



## Business Recommendations

- Implement real-time fraud monitoring.
- Strengthen KYC verification procedures.
- Monitor high-risk merchants continuously.
- Add multi-factor authentication for risky transactions.
- Build machine learning models for automated fraud detection.
